Analyse Technique & Financière

Stratégies de Protection
Excel & Propriété Intellectuelle

"Comment sécuriser notre savoir-faire tout en restant opérationnel ?"

L'enjeu de la décision

Il n’existe pas de solution magique qui soit à la fois totalement hors ligne, inviolable et bon marché. Chaque approche est un compromis technique.

Ce rapport détaille quatre architectures distinctes pour protéger vos fichiers Excel, allant de la méthode "flexible" (Offline) à la méthode "coffre-fort" (Machine Virtuelle).

Critères d'analyse

  • Sécurité du code source
  • Dépendance à Internet
  • Coût d'infrastructure

01. Offline (Confusion)

La Référence Actuelle

Le Principe

L’utilisateur installe le logiciel directement sur son PC. Il utilise Excel normalement. Cependant, les fichiers sont fortement "marqués" (watermark) et le code interne est rendu volontairement confus.

Si quelqu’un copie le fichier, il obtient un document fonctionnel uniquement sur sa machine, mais illisible et inutilisable pour un concurrent ou un tiers.

Avantages
  • • 100% Offline (Zéro internet requis)
  • • Pas de serveur à gérer
  • • Coût très faible
  • • Suffisant pour 95% des abus
Limites
  • • Pas mathématiquement inviolable
  • • Le fichier source est "chez le client"
  • • Un expert motivé peut le casser

02. VM Online (Cloud)

Sécurité Maximale

Le Principe

Le fichier Excel ne quitte jamais nos serveurs sécurisés. Le client envoie ses données via internet, nos serveurs (VM) effectuent les calculs complexes, et renvoient uniquement le résultat final.

C'est le modèle "Coffre-fort distant". Même si le client se fait voler son PC, le voleur ne trouve rien car l'intelligence est dans le Cloud.

Avantages
  • • Rien à voler chez le client
  • • Sécurité absolue (Black Box)
  • • Maintenance facile à distance
  • • Image très "Premium"
Inconvénients
  • • Internet obligatoire pour calculer
  • • Coût récurrent (Abonnement serveur)
Investissement

Serveurs (Récurrent)

03. VM Offline (Locale)

Haute Sécurité / Banques

Le Principe

Nous installons une machine physique (serveur) dédiée dans les locaux du client. C'est exactement comme la solution Cloud, mais le nuage est privé, dans leur bureau.

L'ordinateur de l'utilisateur se connecte au serveur par câble réseau local (LAN). Aucune donnée ne sort du bâtiment.

Avantages
  • • Sécurité physique maximale
  • • Fonctionne sans Internet
  • • Contrôle total des données
Inconvénients
  • • Installation lourde et complexe
  • • Coûts élevés (Matériel + Déplacement)
  • • Maintenance physique requise
Investissement Lourd

Installation (Matériel)

300k - 1M FCFA

Développement

800k - 3.3M FCFA

Maintenance / mois

30k - 150k FCFA

"Réservé aux banques et environnements réglementés."

04. VM User (Locale)

⚠️ Fausse bonne idée

Principe

TSS fournit une machine virtuelle (VirtualBox / VMware) installée directement sur le PC du client. Excel tourne dans cette boîte isolée. L'utilisateur travaille via une interface locale sans internet.

👉 L’idée marketing : "Le moteur est isolé, donc protégé".

❌ Problème fondamental : "Illusion de sécurité"

La VM est sur la machine du client. Le client est administrateur de sa machine.

  • Il peut faire des "Snapshots" (photos de la mémoire).
  • Il peut copier les disques virtuels.
  • Un utilisateur motivé peut extraire Excel et analyser le moteur.
✅ Avantages apparents
  • • 100% Offline
  • • Pas de serveur distant
  • • Facile à déployer techniquement
  • • Rassure certains clients "tech"
⚖️ Verdict Honnête

"Elle complique la vie de l’utilisateur honnête plus qu’elle ne bloque un voleur motivé. À éviter sauf cas très spécifique."
